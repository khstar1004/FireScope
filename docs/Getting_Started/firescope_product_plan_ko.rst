FireScope 제품 기획문서
========================

작성 기준
---------

- 이 문서는 2026-04-13 현재 레포 구현 상태를 기준으로 FireScope를 역으로 정리한 제품 기획문서입니다.
- 사용자에게 노출되는 현재 제품명은 주로 ``FireScope`` 이지만, 코드와 문서 안에는 ``BLADE``, ``Panopticon``, ``Digital Twin`` 같은 이전 명칭이 함께 남아 있습니다.
- 아래 내용은 ``현재 구현으로 확인되는 범위`` 와 ``그 구현이 암시하는 기획적 의미`` 를 함께 정리한 것입니다.


제품 한 줄 정의
---------------

FireScope는 하나의 전장 시나리오를 중심으로 ``배치``, ``시뮬레이션``, ``기록/재생``, ``작전 브리핑형 3D 시뮬레이터``, ``강화학습 설계`` 를 연결하는 전장 시나리오 운영 워크벤치입니다.


제품 정체성
-----------

현재 코드 기준 FireScope는 단일 기능 제품이 아닙니다. 다음 세 층이 결합된 형태입니다.

- ``시나리오 편집기``: 지도 위에 항공기, 함정, 기지, 지상 무기체계, 참조점, 임무를 배치하고 수정합니다.
- ``전술 시뮬레이터``: 시간 진행, 무장 교전, 임무 수행, RTB, 간단한 자동 교전 규칙을 돌려 결과를 확인합니다.
- ``실험/브리핑 플랫폼``: 리플레이, 결과 요약, AI 설명, 강화학습, 3D 쇼룸, 항공 시뮬레이터, 전술 시뮬레이터로 같은 시나리오를 확장합니다.

즉, FireScope의 본질은 ``지도 기반 전장 편집 툴`` 이면서 동시에 ``전술 결과를 설명하고 실험하는 허브`` 에 가깝습니다.


문제 정의
---------

현재 구현이 해결하려는 문제는 다음과 같이 해석할 수 있습니다.

- 전장 배치, 자산 구성, 임무 설계를 빠르게 만들고 즉시 실행해 보고 싶다.
- 단순 지도 편집이 아니라, 시간 흐름 속에서 무장 사용과 결과를 확인하고 싶다.
- 같은 시나리오를 분석, 브리핑, 교육, 강화학습 설계에 재활용하고 싶다.
- 항공/지상/해상 자산을 한 화면에서 다루되, 필요한 순간에는 더 몰입적인 3D 표현으로 전환하고 싶다.
- RL 실험을 위해 별도 데이터 변환 없이 현재 편집 중인 시나리오를 바로 학습 입력으로 쓰고 싶다.


제품 비전
---------

코드 구조가 말하는 FireScope의 핵심 비전은 ``시나리오 단일 원본화`` 입니다.

- 하나의 시나리오 JSON이 편집, 실행, 저장, 리플레이, RL, 일부 3D 시뮬레이터의 출발점이 됩니다.
- 프런트엔드와 Gym 환경이 같은 전장 개념을 공유합니다.
- 결과물은 다시 JSON/JSONL로 저장돼 지도에서 재생하거나 외부 실험에 재사용됩니다.

이 비전은 아직 완전히 통합된 상태는 아니지만, 제품 방향은 분명합니다. FireScope는 단순 시각화 도구보다 ``시나리오 기반 전장 운영 플랫폼`` 으로 설계되고 있습니다.


핵심 사용자
-----------

- ``시나리오 설계자``: 전력 배치, 세력 관계, 참조점, 임무, 교리를 구성하는 사용자
- ``작전 분석가``: 실행 결과, 로그, 전력 소모, 승패 우세를 해석하는 사용자
- ``교육/브리핑 운영자``: 2D 지도와 3D 시뮬레이터를 넘나들며 설명하는 사용자
- ``RL 실험자``: 같은 시나리오를 강화학습 환경에 연결해 정책 비교를 하는 사용자
- ``데모/쇼케이스 사용자``: 자산 모델, 항공 시뮬레이터, 전술 시뮬레이터를 통해 자산과 임무 맥락을 전달하려는 사용자


핵심 가치 제안
-------------

- ``한 번 만든 시나리오를 여러 방식으로 재사용`` 할 수 있습니다.
- ``지도 편집과 실행 결과 확인이 한 흐름`` 안에 있습니다.
- ``시뮬레이션 결과를 기록·재생·요약·설명`` 하는 기능이 내장돼 있습니다.
- ``강화학습 설계`` 가 별도 연구 코드가 아니라 사용자 시나리오와 직접 연결됩니다.
- ``3D 자산 시뮬레이터`` 와 ``전술 시뮬레이터`` 가 브리핑 도구 역할을 합니다.


현재 제품 범위
--------------

현재 구현 기준 FireScope는 아래 기능을 제공합니다.

- 새 시나리오 생성, 기존 데모 불러오기, JSON 업로드, 파일 저장
- 로그인 기반 클라우드 저장/삭제 UI
- 세력 생성/수정, 아군/적군 관계 편집, 세력별 교리 설정
- 항공기, 기지, 함정, 지상 시설, 참조점 배치
- 무장 편집, 수동 공격, RTB, 위치 수정, 복제
- 순찰 임무와 타격 임무 생성/수정/삭제
- 시뮬레이션 단계 진행, 재생, 일시정지, 속도 변경, 실행 취소
- 시뮬레이션 로그, 라이브 코멘터리, 결과 요약/서술
- 기록 저장, 기록 파일 열기, 단계별 리플레이
- 집중포격 모드와 화력 추천
- 자산 3D 쇼룸, 몰입형 자산 시뮬레이터, 전술 3D 시뮬레이터, 전투기/드론 항공 시뮬레이터
- 강화학습 설계 기반 고정 표적 타격 정책 학습
- 로컬 Python 학습 잡 관리와 평가 리플레이 재개방


정보 구조
---------

현재 앱은 해시 라우팅 기준으로 다음 화면과 모듈로 나뉩니다.

- ``메인 지도``: ``ScenarioMap`` 중심의 편집/실행 화면
- ``Asset Experience``: 개별 자산의 3D 소개 화면
- ``Immersive Experience``: 자산군/작전 모드 중심의 몰입형 브리프 화면
- ``Tactical Sim``: 임무 스타터와 3D 전술 HUD가 있는 전술 시뮬레이터 화면
- ``Flight Sim``: 전투기 또는 드론 시점 항공 시뮬레이터
- ``강화학습 설계``: 고정 표적 타격 학습 설계 도구

이 구조는 FireScope가 단일 화면 앱이 아니라 ``지도 편집기 + 시뮬레이터형 서브앱 묶음`` 임을 보여줍니다.


주요 사용자 흐름
----------------

1. 시나리오 작성 및 실행

- 빈 시나리오 또는 프리셋에서 시작합니다.
- 세력과 관계를 정하고 자산을 지도에 배치합니다.
- 참조점과 임무를 만든 뒤 시뮬레이션을 실행합니다.
- 로그, 결과 요약, 승패 우세를 확인합니다.

2. 기록 및 사후 분석

- 시뮬레이션 기록을 JSONL로 저장합니다.
- 저장된 기록을 다시 불러와 단계별 재생합니다.
- 주요 전투 이벤트는 라이브 코멘터리와 결과 요약으로 다시 읽습니다.

3. 집중포격 운영

- 특정 세력에서 집중포격 모드를 활성화합니다.
- 목표 지점을 지정하면 항공/화력/기갑 자산이 해당 축으로 수렴합니다.
- 점령 진행률, 투사량, 비행 중 탄체, 추천 화력안을 확인합니다.
- 공중 관측 3D 화면으로 이어서 상황을 봅니다.

4. 자산 소개 및 브리핑

- 지도에서 자산을 선택한 뒤 3D 시뮬레이터로 이동합니다.
- 자산 메트릭, 연결 모델, 작전 프로필, 비교 모델을 확인합니다.
- 필요하면 전술 3D 시뮬레이터 또는 항공 시뮬레이터로 전환합니다.

5. RL 실험

- 현재 지도 시나리오 또는 업로드 JSON을 강화학습 설계로 넘깁니다.
- 아군 항공기와 적 고정 표적을 선택하고 보상 계수와 알고리즘을 조정합니다.
- 로컬 Python 학습을 실행하고 진행 그래프와 평가 결과를 봅니다.
- 선택된 평가 리플레이를 다시 지도에서 엽니다.


기능 상세
---------

시나리오 편집
~~~~~~~~~~~~~

- 시나리오 이름 변경, 새로 만들기, 불러오기, 파일 저장이 가능합니다.
- 프리셋 시나리오와 업로드 파일, 클라우드 저장 시나리오를 하나의 흐름에서 다룹니다.
- 실행 취소 기능이 있어 편집 실수를 되돌릴 수 있습니다.

세력과 교리
~~~~~~~~~~~

- 세력을 추가하고 이름과 색상을 관리할 수 있습니다.
- 세력 간 아군/적군 관계를 직접 편집합니다.
- 항공기 자동 공격, 추격, RTB 조건, SAM/함정 자동 공격 같은 교리를 세력별로 설정합니다.

자산 체계
~~~~~~~~~

현재 코드상 주요 자산 범주는 다음과 같습니다.

- 항공기
- 기지
- 함정
- 시설

시설은 단순 SAM만이 아니라 다음 프록시형 지상 전력을 포함하는 방향으로 확장돼 있습니다.

- 방공 체계
- 화력 자산
- 기갑 자산
- 지휘/지원 차량

참조점과 무장도 시나리오 핵심 객체로 취급됩니다.

자산 상호작용
~~~~~~~~~~~~~

- 항공기: 수동 공격, RTB, 복제, 위치 수정, 무장 편집, 임무 연결
- 함정: 수동 공격, 위치 수정, 3D 시뮬레이터, 함재기 운용
- 시설: 무장 확인, 위치 수정, 3D 시뮬레이터
- 기지: 위치 수정, 항공기 보관/출격, 3D 시뮬레이터
- 무장: 위치 확인, 3D 시뮬레이터

임무 계획
~~~~~~~~~

- ``Patrol Mission``: 참조점 다각형을 기반으로 초계 구역을 구성합니다.
- ``Strike Mission``: 공격 편대와 표적 묶음을 지정합니다.
- 임무는 특정 세력에 귀속되며, 생성 후 수정과 삭제가 가능합니다.

시뮬레이션 실행
~~~~~~~~~~~~~~

- 1단계 진행, 연속 실행, 일시정지, 속도 변경을 지원합니다.
- 무기 발사, 피격, 탄체 소실, 항공기 손실, 임무 성공/중단 같은 로그가 기록됩니다.
- 일부 적대 행동은 교리 기반 자동 교전으로 처리됩니다.

기록과 리플레이
~~~~~~~~~~~~~~~

- 시나리오 기록은 JSONL 단위로 저장됩니다.
- 파일 크기 제한을 넘으면 분할 다운로드됩니다.
- 기록 재생 시 현재 단계 기준으로 지도와 시간, 로그가 갱신됩니다.
- RL 평가 리플레이도 동일한 지도 리플레이 흐름으로 되돌릴 수 있습니다.

전장 해설과 결과 요약
~~~~~~~~~~~~~~~~~~~~~

- 주요 로그는 라이브 코멘터리 카드로 별도 강조됩니다.
- 시뮬레이션 종료 시 승자, 점수차, 잔존 전력, 최근 로그를 바탕으로 결과 요약이 생성됩니다.
- 결과 서술은 LLM 호출을 시도하고, 실패하면 규칙 기반 fallback 요약을 사용합니다.

집중포격 모드
~~~~~~~~~~~~~

집중포격은 FireScope만의 별도 작전 모드로 보입니다.

- 목표 참조점을 지정합니다.
- 같은 세력의 항공기, 화력 시설, 기갑 시설을 집중축으로 묶습니다.
- 포격 충격도, 우세 축, 비행 중 탄체, 점령 진척을 보여줍니다.
- 화력 추천은 표적 구성, 예상 타격 효과, 무장 종류, 발사 거리, 적합도 점수까지 제공합니다.
- 공중 관측 3D와 연결돼 브리핑용 시각화에 적합합니다.

작전 도우미
~~~~~~~~~~~

- 채팅형 도우미가 현재 시나리오 스냅샷을 읽고 답변합니다.
- 세력 전력, 경고 상태, 임무, 최근 로그가 프롬프트 문맥으로 들어갑니다.
- 용도는 전력 요약, 위험 분석, 임무 제안 같은 작전 보조에 가깝습니다.

3D 자산 시뮬레이터 계층
~~~~~~~~~~~~~~~~~~~~~~

FireScope는 단순 지도 팝업을 넘어서 별도 3D 시뮬레이터 계층을 가지고 있습니다.

- ``Asset Experience``: 개별 자산 메트릭과 3D 모델 소개
- ``Immersive Experience``: 지상, 화력, 방공, 해상, 기지 프로필별 브리핑 화면
- ``Tactical Sim``: 미션 스타터, HUD, 표적 전환, 시점 전환이 있는 전술 시뮬레이터
- ``Flight Sim``: 전투기/드론 조종 시뮬레이터

이 계층은 엄밀한 전장 편집보다 ``자산 설명``, ``브리핑``, ``전시형 시각화`` 에 더 가깝습니다.

강화학습 설계
~~~~~~~~~~~~

강화학습 설계는 현재 제품 안에서 가장 실험적이지만 가장 명확한 연구 기능입니다.

- 현재 환경은 ``blade/FixedTargetStrike-v0`` 하나에 집중돼 있습니다.
- 학습 대상은 ``아군 항공기`` 대 ``적 고정 표적`` 의 협조 타격입니다.
- 시나리오를 읽어 아군 세력, 적 세력, 항공기, 시설/기지/정지 함정을 자동 분석합니다.
- 지원 알고리즘은 ``PPO``, ``A2C``, ``SAC``, ``DDPG``, ``TD3`` 입니다.
- 사용자는 timesteps, eval episodes, curriculum, 보상 계수, 고가치 표적을 조정할 수 있습니다.
- 결과는 성공률, 평균 보상, 생존성, 무장 효율, TOT 품질, seed 변동성 등으로 요약됩니다.
- 학습 산출물은 ``.firescope_rl_jobs/<job-id>/`` 아래에 저장됩니다.


데이터 모델
-----------

FireScope의 핵심 데이터 단위는 ``시나리오 JSON`` 입니다.

- 루트에는 보통 ``currentScenario``, ``currentSideId``, ``selectedUnitId``, ``mapView``, ``focusFireOperation`` 이 들어갑니다.
- ``currentScenario`` 안에는 시간, 세력, 관계, 교리, 항공기, 함정, 시설, 기지, 무장, 참조점, 임무가 들어갑니다.
- 이 구조는 프런트엔드 편집, 리플레이, RL 입력, Gym export에 걸쳐 재사용됩니다.

핵심 객체는 다음과 같습니다.

- ``Scenario``
- ``Side``
- ``Aircraft``
- ``Ship``
- ``Facility``
- ``Airbase``
- ``Weapon``
- ``ReferencePoint``
- ``PatrolMission``
- ``StrikeMission``
- ``Relationships``
- ``Doctrine``


시뮬레이션 모델
---------------

현재 코드가 구현하는 전장 모델은 고정밀 군사 시뮬레이터라기보다 ``경량 전술 시뮬레이션 엔진`` 에 가깝습니다.

- 시간은 1초 간격 기준으로 진행됩니다.
- 이동, 연료 소모, 무장 발사, 피격 판정, 기지 이착륙, 간단한 NPC 교전이 구현돼 있습니다.
- 표적 파괴와 임무 성공이 점수와 로그에 반영됩니다.
- 임무 자동화와 교리 기반 자동 교전이 존재합니다.

동시에 다음 구조적 특징이 있습니다.

- TypeScript 엔진과 Python 엔진이 별도로 존재합니다.
- 프런트엔드는 TypeScript 엔진을 사용합니다.
- RL 환경은 Python ``BLADE`` 엔진을 사용합니다.

이 이중 구조는 기능 확장의 유연성을 주지만, 장기적으로는 동기화 비용과 드리프트 위험을 만듭니다.


기술 아키텍처
-------------

클라이언트
~~~~~~~~~~

- ``React 18``
- ``Vite``
- ``TypeScript``
- ``OpenLayers``
- ``MUI``
- ``Auth0`` UI 연동

클라이언트 안에는 다음 서브시스템이 있습니다.

- ``client/src/game``: TypeScript 시뮬레이션 엔진
- ``client/src/gui``: 지도, 툴바, 팝업, 강화학습 설계, 3D 시뮬레이터
- ``client/src/scenarios``: 데모/전략 프리셋
- ``client/public/flight-sim``: 항공 시뮬레이터 정적 앱
- ``client/public/tactical-sim``: 전술 시뮬레이터 정적 앱

강화학습 백엔드
~~~~~~~~~~~~~~

- ``gym/blade``: Python BLADE 엔진
- ``gym/blade/envs/fixed_target_strike.py``: 현재 RL 핵심 환경
- ``gym/scripts/fixed_target_strike/train.py``: 단일/다중 알고리즘 학습 스크립트
- ``gym/scripts/fixed_target_strike/compare_algorithms.py``: 배치 비교 스크립트

특이점은 FireScope가 별도 웹 백엔드 없이도 로컬 RL을 돌릴 수 있다는 점입니다.

- Vite dev server plugin이 로컬 API처럼 동작합니다.
- 클라이언트에서 RL 학습 요청을 보내면 Python 프로세스를 직접 spawn 합니다.
- 결과 파일을 다시 클라이언트가 읽어 시각화합니다.

시나리오 카탈로그
-----------------

현재 코드상 확인되는 대표 콘텐츠는 다음과 같습니다.

- 빈 시나리오
- 기본 데모
- 집중 영향권 데모
- 전장 데모
- ``Fixed Target Strike RL Demo`` 리플레이 번들
- ``이란 vs 미국 - 호르무즈 봉쇄``
- ``이란 vs 미국 - 알다프라 드론 공습``
- ``한국 vs 북한 - 서해 합동 방어``

전략 프리셋은 단순 장식이 아니라 각 전장 메시지를 갖고 설계돼 있습니다.

- 호르무즈 봉쇄: 해협 통제, 해안 방공, 항모 전단 운용
- 알다프라 드론 공습: 기지 방호, 다층 방공, 드론/미사일 억제
- 서해 합동 방어: 수도권 방어, 서해 함정, 북측 발사대 압박, 드론 ISR

현재 한계와 리스크
-----------------

1. 명칭 혼재

- FireScope, BLADE, Panopticon, Digital Twin 명칭이 함께 남아 있습니다.
- 제품 정체성과 문서 일관성에 혼선을 줍니다.

2. 엔진 이중화

- 프런트와 Gym이 서로 다른 구현체를 유지합니다.
- 기능 추가 시 동기화 비용이 큽니다.

3. RL 범위 제한

- RL은 현재 ``고정 표적 타격`` 서브태스크에 한정됩니다.
- 지상 기동전, 해상 maneuver, 장기 캠페인 최적화는 직접 다루지 못합니다.

4. 서버 최소화

- 인증 헤더는 받지만 검증하지 않습니다.
- 시나리오 저장은 최신 5개만 유지하는 단순 구조입니다.
- 협업, 버전 관리, 권한 제어는 없습니다.

5. 문서 드리프트 가능성

- 일부 문서는 현재 코드와 정확히 일치하지 않을 수 있습니다.
- 예를 들어 제품명과 LLM 설정 설명은 최신 코드 기준으로 다시 정리할 필요가 있습니다.

6. 3D 시뮬레이터의 상태 동기화 한계

- 3D 쇼룸과 전술 시뮬레이터는 브리핑/시뮬레이션 중심입니다.
- 메인 시뮬레이션 전체 상태를 완전히 복제하는 범용 3D 디지털 트윈이라고 보기는 어렵습니다.

7. 외부 키 의존성

- 지도 화면은 환경 변수와 외부 타일 서비스 설정에 의존합니다.


권장 제품 지표
--------------

아래 항목은 현재 코드에 직접 구현돼 있지는 않지만, 이 제품의 성숙도를 관리하기 위해 권장되는 KPI입니다.

- 시나리오 생성 후 첫 실행까지 걸린 시간
- 저장된 시나리오의 재불러오기 비율
- 기록 생성 후 재생까지 이어지는 비율
- RL 학습 시작 대비 정상 완료 비율
- RL 리플레이를 다시 지도에서 여는 비율
- 자산 선택 후 3D 시뮬레이터 진입 비율
- 결과 요약 창에서 AI 서술 생성 성공 비율

최종 정리
---------

현재 코드 기준 FireScope는 ``전장 시나리오 편집기`` 에서 시작해 ``전술 시뮬레이터``, ``리플레이/해설 도구``, ``3D 브리핑 시뮬레이터``, ``강화학습 설계`` 까지 연결하는 복합 제품입니다.

완성형 대규모 C2 플랫폼이나 고정밀 군용 시뮬레이터라기보다는, ``시나리오를 빠르게 만들고 실험하고 설명하는 전장 실험 워크벤치`` 로 정의하는 것이 가장 정확합니다.
