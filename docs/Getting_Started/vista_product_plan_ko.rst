========================

---------

- 이 문서는 ``2026-04-21`` 현재 레포 구현 상태를 기준으로 VISTA를 역으로 정리한 현행 PRD입니다.
- 이전 ``2026-04-13`` 버전 대비 특히 크게 달라진 축은 ``가이드형 시나리오 작성``, ``집중포격 운용 패널과 AI 재정렬``, ``전장 3D 관전``, ``RL Lab의 멀티 알고리즘 비교``, ``Commander 자원·배치 탐색`` 입니다.
- 검토 기준 구현은 주로 ``client/src/App.tsx``, ``client/src/gui/map/ScenarioMap.tsx``, ``client/src/gui/map/toolbar/Toolbar.tsx``, ``client/src/gui/fires/FocusFireDockPanel.tsx``, ``client/src/gui/flightSim/FlightSimPage.tsx``, ``client/src/gui/rl/RlLabPage.tsx``, ``client/src/gui/rl/RlLabCommanderPanel.tsx``, ``client/scripts/createRlDevServerPlugin.ts``, ``client/src/game/Game.ts``, ``gym/scripts/fixed_target_strike/train.py``, ``gym/scripts/fixed_target_strike/commander_optimize.py`` 입니다.

제품 한 줄 정의
---------------

VISTA는 하나의 시나리오를 기준으로 ``가이드형 자산 배치``, ``경량 전술 시뮬레이션``, ``기록/재생``, ``전장 3D 관전 및 브리핑``, ``RL/Commander 실험`` 을 연결하는 시나리오 운영 워크벤치입니다.


제품 정체성과 현재 포지셔닝
---------------------------

현재 구현 기준 VISTA는 단일 기능 제품이 아닙니다. 다음 여섯 층이 결합된 복합 제품입니다.

- ``시나리오 저작 도구``: 항공기, 기지, 함정, 시설, 지상군, 참조점, 임무를 지도에서 직접 배치하고 수정합니다.
- ``경량 전술 시뮬레이터``: 시간 흐름, 자동 교전, 무장 발사, 피격, 연료, 임무 수행, 종료 판정을 빠르게 돌려 결과를 확인합니다.
- ``가이드형 작성 경험``: Guide Rail, 빠른 배치, 자산 프리뷰, 시작 시점 2D/3D 선택 등으로 빈 시나리오에서 첫 실행까지의 마찰을 줄입니다.
- ``화력 운용 도구``: 집중포격 모드, 목표 지정, 권장 화력안, BDA 요약, 운영자 피드백 기반 AI 재정렬을 제공합니다.
- ``브리핑·관전 계층``: 자산 쇼룸, 몰입형 브리핑, 전술 3D, 전장 3D 관전, 공중 관측을 같은 시나리오와 연결합니다.
- ``실험 워크스페이스``: 고정표적 타격 RL 학습, 체크포인트 리플레이, 멀티 알고리즘 비교, 지휘관 자원·배치 탐색을 지원합니다.

즉, VISTA의 본질은 단순 지도 도구가 아니라 ``하나의 시나리오를 만들고, 돌리고, 설명하고, 비교 실험하는 허브`` 에 가깝습니다.


VISTA가 해결하려는 문제
---------------------------

현재 구현이 겨냥하는 문제는 다음과 같이 정리할 수 있습니다.

- 복합 전력 구성을 빠르게 만들어 바로 실행해 보고 싶다.
- 단순 배치 결과가 아니라 시간 흐름 속 교전, 손실, 무장 사용, 우세 변화를 보고 싶다.
- 한 번 만든 시나리오를 분석, 브리핑, 데모, 교육, 정책 비교에 반복 재사용하고 싶다.
- 2D 설계 경험과 3D 설명 경험을 같은 시나리오에서 자연스럽게 오가고 싶다.
- 화력 운용과 표적 우선순위를 사람 중심으로 조정하되, 추천과 AI 재정렬의 도움도 받고 싶다.
- RL 실험과 COA 탐색을 위해 현재 시나리오를 별도 포맷 변환 없이 바로 실험 입력으로 쓰고 싶다.

VISTA의 핵심 접근은 여전히 ``시나리오 단일 원본화`` 이지만, 이제는 그 위에 ``가이드형 작성`` 과 ``설명 가능한 실험`` 이 함께 올라간 상태로 보는 것이 더 정확합니다.


제품 비전
---------

VISTA의 장기 비전은 다음과 같이 정리할 수 있습니다.

``하나의 시나리오를 설계, 실행, 관전, 결과 해설, 실험의 공통 원본으로 삼는 시나리오 기반 운영 플랫폼``

현재 구현이 드러내는 제품 원칙은 다음과 같습니다.

- ``시나리오 단일 원본``: 편집, 저장, 실행, 기록, 리플레이, RL 입력이 같은 시나리오 엔벌로프를 공유합니다.
- ``가이드형 작성``: 빈 지도에서 시작해도 빠른 배치와 단계 가이드를 통해 첫 실행까지 도달할 수 있어야 합니다.
- ``설명 가능한 결과``: 로그, 코멘터리, BDA, 3D 관전, 요약을 통해 결과를 다시 읽을 수 있어야 합니다.
- ``사람-중심 실험``: AI와 RL은 자동 통제보다 추천, 비교, 후보 축소를 돕는 방향이어야 합니다.
- ``2D-3D 연속성``: 설계는 2D 중심으로, 설명과 관전은 3D 중심으로 이어져야 합니다.
- ``로컬 우선 실험성``: 무거운 중앙 서버 없이도 로컬에서 학습 잡과 후보 탐색을 돌릴 수 있어야 합니다.


핵심 사용자
-----------

- ``시나리오 설계자``: 세력, 자산, 관계, 교리, 참조점, 임무를 조합해 시나리오를 만드는 사용자
- ``화력 운용자``: 집중포격 목표를 지정하고 화력 추천안, 요망 효과, 공중 관측을 활용하는 사용자
- ``작전 분석가``: 로그, 결과 요약, BDA, 리플레이, 체크포인트 성과를 읽는 사용자
- ``교육·브리핑 운영자``: 2D 지도와 3D 체험/관전 화면을 전환하며 설명하는 사용자
- ``RL 실험자``: 같은 시나리오를 RL 학습과 평가 리플레이로 연결해 정책을 비교하는 사용자
- ``COA 탐색 사용자``: 자산 조합과 초기 배치를 바꿔가며 더 나은 후보안을 찾는 사용자


핵심 가치 제안
-------------

- ``한 번 만든 시나리오를 여러 층에서 재사용`` 할 수 있습니다.
- ``가이드형 작성`` 으로 빈 시나리오에서 첫 실행까지의 시간을 줄입니다.
- ``집중포격-분석-공중 관측`` 이 같은 흐름으로 이어져 화력 운용 설명력이 높습니다.
- ``전장 3D 관전`` 으로 현재 시나리오의 유닛, 미사일, 최근 교전을 브리핑형 시점에서 다시 읽을 수 있습니다.
- ``RL Lab + Commander`` 로 정책 비교와 후보 탐색을 같은 제품 안에서 수행할 수 있습니다.


현재 제품 범위
--------------

시나리오 라이프사이클
~~~~~~~~~~~~~~~~~~~~~

- 새 시나리오 생성, 이름 변경, 프리셋 불러오기, JSON 업로드, 로컬 파일 저장을 지원합니다.
- 서버 모드에서는 로그인 후 클라우드 저장/목록/삭제 UI를 지원합니다.
- 클라우드 저장은 현재 단순 저장소이며 최신 ``5개`` 시나리오만 유지합니다.
- 실행 취소와 시나리오 재시작 흐름이 기본 포함됩니다.

가이드형 작성 경험
~~~~~~~~~~~~~~~~~

- ``Experience Guide Rail`` 이 ``자산 선택 -> 자산 배치 -> 실행 -> 3D 관전 -> 결과 분석`` 단계를 보여줍니다.
- 자산 혼합별 빠른 배치 버튼과 공백 알림을 제공해 초반 작성 동선을 유도합니다.
- 자산 배치 전 3D 모델 또는 개념 프리뷰를 보여주는 ``Asset Placement Preview`` 가 있습니다.
- 시나리오가 시작 경계에 있고 일시정지 상태일 때는 ``2D`` 또는 ``3D`` 시작 모드를 선택할 수 있습니다.

세력, 관계, 교리
~~~~~~~~~~~~~~~~

- 세력 생성/수정/삭제, 색상 변경, 아군/적군 관계 편집이 가능합니다.
- 세력별 교리 설정이 가능하며 자동 교전, 추격, RTB, 방공 반응 등에 영향을 줍니다.
- 현재 세력 기준으로 자산 배치와 필터링이 이뤄집니다.

자산 체계
~~~~~~~~~

현재 핵심 자산 범주는 다음과 같습니다.

- ``Aircraft``
- ``Airbase``
- ``Ship``
- ``Facility``
- ``Army``
- ``Weapon``
- ``ReferencePoint``

시설과 지상군은 단순 SAM만이 아니라 방공, 포병, 기갑, 지휘/지원 차량 프록시까지 포함하는 방향으로 확장되어 있습니다.

배치와 편성 기능
~~~~~~~~~~~~~~~~

- 항공기, 함정, 지상군, 시설은 경로 지정과 위치 수정, 일부 자동 공격/수동 공격을 지원합니다.
- 무장 수량 편집, 기지/함정 탑재 항공기 관리, RTB, 복제, 텔레포트가 가능합니다.
- 시설은 ``배치 그룹`` 으로 묶어 저장할 수 있으며, 드래그 선택 후 그룹 이동/삭제가 가능합니다.
- 체력/방어 기반 상태가 지도 스타일과 카드 UI에 반영됩니다.

임무 계획
~~~~~~~~~

- ``Patrol Mission``: 참조점 다각형을 기반으로 초계 구역을 설정합니다.
- ``Strike Mission``: 공격 편대와 표적 묶음을 지정합니다.
- 임무 생성, 수정, 삭제, 할당 상태 편집이 가능합니다.

시뮬레이션 실행
~~~~~~~~~~~~~~

- 시뮬레이션은 기본적으로 ``1초`` 단위로 상태를 갱신합니다.
- 1단계 진행, 연속 실행, 일시정지, 속도 변경을 지원합니다.
- 이동, 연료 소모, 자동 방어, 무장 발사, 피격/격추, 임무 갱신, 종료 판정이 포함됩니다.
- 종료 조건은 시간 제한, 활성 세력 소멸, 단일 세력 잔존 등으로 구성됩니다.

기록과 리플레이
~~~~~~~~~~~~~~~

- 시뮬레이션 상태를 ``JSONL`` 단위로 기록하고 다시 불러올 수 있습니다.
- 기록 재생 시 타임라인 슬라이더, 이전/다음 시점, 자동 재생을 지원합니다.
- 기록 문자열이 길어지면 분할 다운로드가 일어나는 구조를 유지하고 있습니다.
- RL 평가 리플레이와 체크포인트 리플레이도 동일한 지도 리플레이 흐름으로 연결됩니다.

라이브 해설과 결과 요약
~~~~~~~~~~~~~~~~~~~~~~~

- 주요 이벤트는 라이브 코멘터리 카드로 강조됩니다.
- 종료 시 결과 요약과 BDA 성격의 설명이 생성됩니다.
- LLM 기반 서술을 시도하되 실패 시 규칙 기반 fallback을 유지합니다.

대화형 도우미
~~~~~~~~~~~~

- ``VISTA Assistant`` 가 현재 시나리오 스냅샷, 세력 상태, 임무, 최근 로그를 바탕으로 질의응답을 수행합니다.
- 용도는 자동 실행보다는 상황 요약, 위험 해석, 임무/화력 제안 같은 이해 보조에 가깝습니다.
- LLM 제공자는 환경 변수 설정에 따라 교체 가능하며, 키 미설정 시 친절한 오류를 반환합니다.

집중포격 운용
~~~~~~~~~~~~

집중포격은 현재 VISTA에서 가장 뚜렷하게 제품화된 별도 작전 모드입니다.

- 목표 참조점을 지정하면 항공, 포병, 기갑 자산이 같은 작전 축으로 수렴합니다.
- ``작전 / 추천 / AI`` 탭을 가진 전용 도크 패널이 제공됩니다.
- 요망 효과 override, 충격량, 점령 진행률, 비행 중 탄체, 우세 축 요약을 보여줍니다.
- 권장 화력안에 대해 운영자 피드백을 기록할 수 있습니다.
- 추천 기록으로 로컬 ``AI 재정렬 모델`` 을 학습, 초기화, 내보내기/불러오기 할 수 있습니다.
- 추천 텔레메트리는 ``JSONL`` 과 ``CSV`` 로 추출할 수 있습니다.
- 같은 작전을 ``공중 관측 3D`` 로 바로 전환할 수 있습니다.

3D 경험 계층
~~~~~~~~~~~~

현재 3D 계층은 단일 페이지가 아니라 목적별 서브앱 묶음입니다.

- ``Asset Experience``: 자산 메트릭과 연결된 3D 모델을 보여주는 쇼룸
- ``Immersive Experience``: 프로필별 브리핑 화면과 commander's intent 중심 뷰
- ``Tactical Sim``: 실제 3D 지도 위 전술 HUD와 모델 집중/작전 투입 흐름
- ``AirCombatOverlay``: 지도에서 곧바로 여는 항공 전술 오버레이
- ``Flight Sim``: 조종형 체험 + 전장 3D 관전 + 집중포격 공중 관측

특히 ``전장 3D 관전`` 은 단순 비행 체험이 아니라 현재 시나리오의 유닛, 무장 궤적, 최근 교전선, 핫스팟을 실시간으로 읽는 관전자 모드로 진화한 상태입니다.

RL Lab
~~~~~~

RL Lab은 더 이상 단순한 단일 학습 페이지로 보기 어렵습니다.

- 대상 환경은 기본적으로 ``vista/FixedTargetStrike-v0`` 입니다.
- 학습 알고리즘은 ``PPO``, ``A2C``, ``SAC``, ``DDPG``, ``TD3`` 를 지원합니다.
- ``smoke``, ``quick``, ``standard``, ``extended``, ``curriculum`` 프리셋이 있습니다.
- 보상 계수, seed, eval seed count, progress evaluation, bootstrap 단계 등을 조정할 수 있습니다.
- 결과는 success rate, mean reward, survivability, weapon efficiency, time to ready, TOT quality 등으로 요약됩니다.
- 체크포인트 리플레이를 지도에서 다시 열 수 있고, 최종 모델/요약/리플레이 아티팩트를 다운로드할 수 있습니다.
- 멀티 시드 평가와 알고리즘 leaderboard, retained model archive를 보여줍니다.

Commander 자원·배치 탐색
~~~~~~~~~~~~~~~~~~~~~~~~

현재 RL Lab 안에는 별도의 ``지휘관 자원·배치 최적화`` 패널이 통합되어 있습니다.

- 전술 RL을 내부 평가기로 사용하면서 자산 조합과 초기 배치를 바꿔 COA를 비교합니다.
- 후보 탐색 축은 아군 자산 subset, 거리 스케일, 방위각 오프셋, 편대 간격, 고가치 표적 우선순위입니다.
- ``smoke``, ``quick``, ``standard`` 프리셋과 ``dry-run`` 모드를 제공합니다.
- 결과는 candidate leaderboard, 선택된 후보 시나리오, 선택 후보 평가 리플레이로 다시 확인할 수 있습니다.
- 이 기능은 제품이 단순 RL 학습기에서 ``COA 탐색 보조 도구`` 로 넓어졌음을 보여줍니다.

시나리오 카탈로그
~~~~~~~~~~~~~~~~~

현재 코드상 주요 시작점은 다음과 같습니다.

- 빈 시나리오
- 기본 데모와 지상군 데모
- 집중포격 경제성/훈련 데모
- RL 첫 성공 데모와 전투 최적화 데모
- ``이란 vs 미국 - 호르무즈 봉쇄``
- ``이란 vs 미국 - 알다프라 드론 공습``
- ``한국 vs 북한 - 서해 합동 방어``

전략 프리셋은 이름만 있는 샘플이 아니라 ``designIntent`` 와 ``assetHighlights`` 를 함께 갖는 콘텐츠형 시나리오입니다.

아직 강한 범위로 보기 어려운 것
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- 협업 편집, 권한 제어, 정식 버전 관리
- 중앙 서버 기반의 다중 사용자 운영
- 센서, 전자전, 물류, 증원까지 포함한 고정밀 시뮬레이션
- 범용 전략 AI 또는 장기 캠페인 최적화
- 메인 시뮬레이션을 완전 복제하는 실시간 디지털 트윈 수준 3D


정보 구조
---------

현재 앱은 ``메인 편집기 + 체험형 서브앱 + 실험 워크스페이스`` 구조입니다.

- ``메인 지도``: ScenarioMap 중심의 작성/실행/리플레이 허브
- ``Guide Rail + Toolbar``: 작성 가이드, 실행 제어, 시나리오 관리, VISTA Assistant, Focus Fire
- ``Asset Experience``: 개별 자산 쇼룸
- ``Immersive Experience``: 브리핑형 자산/프로필 경험
- ``Tactical Sim``: 전술 3D 런타임
- ``Flight Sim``: 조종형 체험과 전장 3D 관전
- ``RL Lab``: 전술 RL 학습과 Commander 탐색
- ``Minimal Server``: 시나리오 저장소와 맵 설정 반환
- ``Vite Local RL API``: 로컬 Python 잡 실행과 아티팩트 전달

즉, VISTA는 더 이상 단일 화면 앱이 아니라 ``시나리오 허브형 제품군`` 으로 보는 편이 정확합니다.


주요 사용자 흐름
----------------

1. 가이드형 시나리오 작성

- 빈 시나리오 또는 프리셋에서 시작합니다.
- Guide Rail과 빠른 배치를 이용해 아군/적군 자산을 놓습니다.
- 세력, 교리, 참조점, 임무를 정리한 뒤 실행합니다.

2. 집중포격 운용

- 세력 선택 후 집중포격 모드를 켭니다.
- 목표 지점을 찍고 추천 화력안과 요망 효과를 확인합니다.
- 필요하면 운영자 피드백을 남기고 AI 재정렬을 학습합니다.
- 공중 관측 3D로 작전 축을 이어서 관찰합니다.

3. 기록과 사후 분석

- 기록을 JSONL로 저장합니다.
- 리플레이 타임라인으로 특정 시점과 로그를 다시 확인합니다.
- 결과 요약과 BDA형 설명으로 사후 해석을 수행합니다.

4. 브리핑과 3D 관전

- 자산 단위로 쇼룸/브리핑에 진입하거나 시나리오 전체를 전장 3D 관전으로 엽니다.
- 최근 교전, 위협 상위 유닛, 미사일 궤적을 3D에서 다시 읽습니다.

5. RL 학습

- 현재 시나리오 또는 업로드 JSON을 RL Lab으로 보냅니다.
- 아군, 적 표적, 고가치 표적, 보상 계수, 알고리즘을 설정합니다.
- 학습 후 체크포인트와 최종 결과를 비교하고 필요하면 지도에서 리플레이를 엽니다.

6. Commander COA 탐색

- 같은 시나리오를 기준으로 자산 조합과 초기 배치를 여러 후보안으로 확장합니다.
- 전술 RL 평가 결과를 바탕으로 후보안 순위를 봅니다.
- 선정된 후보 리플레이를 다시 메인 지도에서 검토합니다.


데이터 모델

VISTA의 핵심 데이터 단위는 여전히 ``시나리오 JSON 엔벌로프`` 입니다.

현재 export 구조에는 보통 다음이 포함됩니다.

- ``currentScenario``
- ``currentSideId``
- ``selectedUnitId``
- ``mapView``
- ``simulationLogs``
- ``focusFireOperation``
- ``focusFireRecommendationTelemetry``
- ``focusFireRerankerEnabled``
- ``focusFireRerankerModel``

``currentScenario`` 내부 핵심 객체는 다음과 같습니다.

- ``Scenario``
- ``Side``

- ``Army``
- ``Airbase``
- ``Facility``
- ``Ship``
- ``Weapon``
- ``ReferencePoint``

- ``StrikeMission``

여기서 중요한 점은 두 가지입니다.

- ``metadata`` 가 시설 배치 그룹 같은 운영 보조 정보를 포함하기 시작했습니다.
- Focus Fire 관련 상태와 AI 추천 텔레메트리까지 시나리오 엔벌로프와 함께 저장/복원됩니다.

즉, VISTA의 데이터 모델은 이제 순수 지형 배치 데이터라기보다 ``시나리오 운영 상태`` 까지 포함하는 쪽으로 넓어졌습니다.


시뮬레이션 모델
---------------

현재 코드가 구현하는 모델은 ``경량 전술 시뮬레이션 엔진`` 으로 보는 것이 적절합니다.

- 시간은 기본적으로 1초씩 진행됩니다.
- 이동, 연료, 자동 방어, 공대공 교전, 무장 비행, 명중/실패, 임무 업데이트가 구현돼 있습니다.
- 포인트 축적, 종료 판정, 최근 이벤트 추적이 포함됩니다.
- 집중포격 같은 상위 운용 모드는 이 엔진 위에서 규칙 기반으로 돌아갑니다.

구조적으로는 여전히 ``이중 엔진`` 상태입니다.

- 프런트엔드 실행은 TypeScript ``Game`` 엔진이 담당합니다.
- RL과 Commander 평가는 Python ``blade`` 엔진을 사용합니다.

이 구조는 빠른 실험에는 유리하지만, 장기적으로는 드리프트 관리가 핵심 과제입니다.


기술 아키텍처
-------------

클라이언트
~~~~~~~~~~

- ``React 18``
- ``Vite``
- ``TypeScript``
- ``OpenLayers``
- ``MUI``
- ``Auth0`` UI 연동

주요 서브시스템은 다음과 같습니다.

- ``client/src/game``: TypeScript 시뮬레이션 엔진
- ``client/src/gui/map``: 메인 지도, 툴바, 리플레이, Guide Rail, Focus Fire
- ``client/src/gui/experience``: 쇼룸, 브리핑, 전술 3D

- ``client/src/gui/rl``: RL Lab과 Commander 패널
- ``client/public/flight-sim`` / ``client/public/tactical-sim``: 정적 3D 런타임

로컬 RL 오케스트레이션
~~~~~~~~~~~~~~~~~~~~~

``client/scripts/createRlDevServerPlugin.ts`` 가 Vite 서버 안에서 로컬 RL API처럼 동작합니다.

- ``/api/rl/fixed-target-strike/*``: RL 학습 capabilities, job, artifact
- ``/api/rl/commander/*``: Commander capabilities, job, artifact
- 로컬 Python 프로세스를 직접 spawn 하며 job metadata를 파일 시스템에 저장합니다.
- 작업 디렉터리는 ``.vista_rl_jobs`` 와 ``.vista_rl_commander_jobs`` 입니다.

- ``GET /health``

- ``GET /api/v1/scenarios``
- ``POST /api/v1/scenarios``
- ``DELETE /api/v1/scenarios/:scenarioId``

실질 역할은 건강 상태 확인, 타일 설정 반환, 시나리오 저장/목록/삭제에 가깝습니다.

품질 신호와 현재 성숙도

VISTA는 단순 데모를 넘어 기능이 누적된 제품형 코드베이스로 보는 편이 적절합니다.

- 엔진, 유닛, 교전, 종료 판정, 기록/재생 테스트가 넓게 분포합니다.
- Focus Fire, BDA 요약, VISTA Assistant, Guide Rail, 3D 라우팅, Battle Spectator 테스트가 존재합니다.
- RL Lab, Commander 패널, 서버 엔드포인트, 단위 DB 정합성 테스트도 포함됩니다.
- 전략 프리셋이 단순 JSON 모음이 아니라 design intent를 가진 시나리오 카탈로그로 발전했습니다.

즉, VISTA는 ``실험형이지만 지속적으로 제품화가 진행 중인 상태`` 로 평가하는 것이 타당합니다.


현재 한계와 리스크
-----------------

1. 브랜드와 명칭 혼재

- VISTA가 주 제품명이지만, UI의 ``VISTA`` 표기와 레거시 명칭이 여전히 섞여 있습니다.
- 대외 문서와 제품 정체성의 일관성을 해칩니다.

2. 맵 설정 이중성

- 서버는 ``/api/v1/map-config`` 를 제공하지만, 메인 지도는 여전히 클라이언트 환경 변수의 MapTiler 키가 없으면 렌더링되지 않습니다.
- 배포/운영 관점에서 설정 책임이 분산돼 있습니다.

3. 엔진 이중화

- TypeScript와 Python 엔진이 병존합니다.
- Focus Fire, 점수 판정, RL 입력 계약이 늘어날수록 정합성 관리 비용이 커집니다.

4. RL/Commander 범위 제한

- 기능은 크게 넓어졌지만 여전히 ``FixedTargetStrike`` 평가기를 중심으로 묶여 있습니다.
- 넓은 지상기동전, 해상 기동전, 장기 캠페인 최적화로 일반화되었다고 보기는 어렵습니다.

5. 서버 최소화

- 인증 헤더는 전달되지만 서버에서 실질 검증을 수행하지 않습니다.
- 저장소는 최신 5개 보관 외에 버전 관리, 협업, 권한 제어가 없습니다.

6. 3D 계층의 상태 동기화 한계

- 전장 3D 관전은 매우 강해졌지만, 메인 시뮬레이션의 모든 상태와 조작을 완전 대체하는 범용 디지털 트윈은 아닙니다.
- 일부 3D 런타임은 정적 자산 동기화에 의존합니다.

7. 로컬 실행 의존성

- RL과 Commander는 로컬 Python 환경과 파일 시스템 쓰기 권한에 크게 의존합니다.
- 팀 협업형 운영 도구로 바로 전환되기에는 운영 모델이 아직 개인 워크스테이션 중심입니다.

8. 문서 드리프트

- 사용 설명서와 일부 README는 현재 기능 수준을 충분히 반영하지 못합니다.
- 제품 문서 체계 재정렬이 필요합니다.

최종 정리
---------

현재 구현 기준 VISTA는 ``시나리오 편집기`` 에서 출발했지만, 이제는 ``가이드형 작성 도구``, ``경량 전술 시뮬레이터``, ``집중포격 운용 도구``, ``전장 3D 관전 계층``, ``RL/Commander 실험 워크스페이스`` 까지 연결하는 복합 제품입니다.

가장 정확한 정의는 다음과 같습니다.

``VISTA는 복합 시나리오를 빠르게 만들고, 실행하고, 기록하고, 3D로 설명하고, 다시 정책과 배치를 비교 실험할 수 있게 해주는 시나리오 기반 운영 워크벤치입니다.``

현재 제품의 가장 큰 강점은 기능 하나의 깊이보다도 ``하나의 시나리오를 작성 -> 실행 -> 화력 운용 -> 관전 -> 리플레이 -> RL/Commander 실험`` 으로 연결하는 구조적 일관성에 있습니다.

반대로 가장 큰 과제는 ``브랜드 정리``, ``TS/Python 계약 정렬``, ``서버 고도화``, ``3D 동기화 충실도 향상`` 입니다.
