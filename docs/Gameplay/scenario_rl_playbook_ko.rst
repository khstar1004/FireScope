시나리오 기반 강화학습 활용 가이드
=================================

개요
----

이 문서는 FireScope 시나리오 프리셋을 현재 RL Lab에 어떻게 연결해서 활용할지 정리합니다.
핵심은 다음 두 가지입니다.

* 지금 바로 가능한 것은 ``고정된 배치 위에서 항공 타격 정책을 학습`` 하는 것입니다.
* 아직 직접 되지 않는 것은 ``전차, 드론, 함정, 방공망의 초기 배치 자체를 RL이 바로 재배치`` 하는 것입니다.

따라서 현재 구조에서 강화학습은 "전장 전체 자동 배치기"보다는
"지금 만든 시나리오를 더 잘 운용하는 정책 탐색기"로 이해하는 것이 정확합니다.


현재 코드 기준으로 되는 것과 안 되는 것
------------------------------------

지원되는 것
~~~~~~~~~~~

* 지도에서 현재 시나리오를 그대로 RL Lab으로 넘길 수 있습니다.
* RL Lab은 시나리오를 읽어서 세력, 아군 항공기, 적 고정 표적을 자동 분석합니다.
* 학습 환경은 ``blade/FixedTargetStrike-v0`` 이며, 고정 표적 타격 성공률을 높이는 문제를 풉니다.
* 같은 시나리오에서 알고리즘, 보상 계수, 표적 묶음을 바꿔 비교할 수 있습니다.

직접 지원되지 않는 것
~~~~~~~~~~~~~~~~~~~~~

* 전차, 장갑차, 드론, 함정의 초기 위치를 RL이 바로 다시 놓는 기능
* 전장 전체 승패를 끝까지 직접 최적화하는 전략 환경
* 장기 캠페인, 증원 파동, 다단계 임무 전환을 포함한 계층형 의사결정

중요한 제약
~~~~~~~~~~~

현재 RL Lab의 자동 분석 기준은 다음과 같습니다.

* 아군 controllable 자산: ``scenario.aircraft`` 에 들어 있는 항공기
* 적 표적 후보: ``facilities``, ``airbases``, 그리고 ``speed == 0`` 인 정지 함정
* 자동 추천 기본 묶음: 아군 4대, 표적 4개, 고가치 표적 2개

즉, 지상 전차나 이동 중인 함정은 현재 RL의 직접 표적이 아닙니다.
하지만 이 자산들은 여전히 전장 맥락을 만들기 때문에, 배치 실험에는 중요한 간접 변수로 활용할 수 있습니다.


시나리오별 RL 적합도
-------------------

한국 vs 북한 - 서해 합동 방어
~~~~~~~~~~~~~~~~~~~~~~~~~~~

현재 가장 RL에 잘 맞는 프리셋입니다.

* 북한 측에 발사대, 방공망, 전진 활주로 같은 고정 표적이 많습니다.
* 한국 측도 다수의 전술기를 동시에 운영할 수 있어 편대-표적 배분 학습 효과가 잘 드러납니다.
* 현재 배치는 한국 우세로 조정돼 있으므로, RL의 목적은 "어떻게 더 안정적으로 이기느냐"에 가깝습니다.

이란 vs 미국 - 알다프라 드론 공습
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

두 번째로 적합한 프리셋입니다.

* 대형 공군기지, 방공대, 발사대, 분산 활주로가 많아 고정 표적 타격 과제와 잘 맞습니다.
* 기지 생존성과 표적 우선순위 실험에 유용합니다.

이란 vs 미국 - 호르무즈 봉쇄
~~~~~~~~~~~~~~~~~~~~~~~~~~~

현재 RL 적합도는 상대적으로 낮습니다.

* 해상 교전 비중이 크고, 이동 함정이 많습니다.
* RL Lab은 정지 함정만 표적으로 읽기 때문에 전장 핵심을 완전히 반영하지 못합니다.


한국 vs 북한 시나리오 추천 워크플로
--------------------------------

목표 설정
~~~~~~~~

이 시나리오에서 RL로 바로 묻기 좋은 질문은 다음입니다.

* 어떤 전술기 4대를 먼저 내보내는 편이 가장 안정적인가
* 발사대와 SAM을 어떤 순서로 눌러야 성공률이 올라가는가
* 활주로 타격을 먼저 할지, 방공 제압을 먼저 할지
* 공격성을 높였을 때 손실과 성공률이 어떤 비율로 바뀌는가

권장 시작 세팅
~~~~~~~~~~~~~

처음 실험은 아래처럼 단순하게 잡는 편이 좋습니다.

* controllable side: ``한국``
* target side: ``북한``
* 시작 아군 후보:

  * ``ROK Ghost 19`` ``F-35A Lightning II``
  * ``ROK Boramae 14`` ``KF-21 Boramae``
  * ``ROK Viper 20`` ``KF-16``
  * ``ROK Eagle 21`` ``F-15 Eagle``

* 시작 표적 후보:

  * ``DPRK Tactical Launcher Belt``
  * ``DPRK Long-Range SAM Proxy``
  * ``DPRK Medium SAM Proxy``
  * ``Haeju Forward Strip`` 또는 ``Kaesong Forward Strip``

* 고가치 표적:

  * ``Haeju Forward Strip``
  * ``Kaesong Forward Strip``

실험 순서
~~~~~~~~

1. 지도에서 ``한국 vs 북한 - 서해 합동 방어`` 를 불러옵니다.
2. 필요하면 배치를 저장해서 ``A안``, ``B안``, ``C안`` 처럼 복제합니다.
3. ``RL Lab`` 으로 이동해 현재 시나리오를 넘깁니다.
4. 자동 추천을 확인하되, 위 권장 아군/표적 묶음으로 먼저 기준선을 만듭니다.
5. 알고리즘은 ``PPO`` 단독으로 한 번 돌립니다.
6. 기준선이 생기면 같은 시나리오에서 ``PPO + A2C`` 비교를 돌립니다.
7. ``success_rate`` 와 리플레이를 보고 어떤 표적 순서가 안정적인지 확인합니다.
8. 필요한 경우 표적을 활주로 중심형, 방공 제압형으로 나눠 다시 실험합니다.

권장 학습 파라미터
~~~~~~~~~~~~~~~~~

처음에는 과도하게 크게 잡지 말고 아래 정도에서 시작하는 것이 무난합니다.

* ``timesteps``: ``4096`` 또는 ``8192``
* ``evalEpisodes``: ``5``
* ``progressEvalFrequency``: ``512``
* ``progressEvalEpisodes``: ``2``

보상 계수 방향
~~~~~~~~~~~~~

한국 우세 배치를 더 안정적으로 굳히고 싶다면 아래 방향이 잘 맞습니다.

* ``Loss Penalty`` 는 높게 둡니다.
* ``Success Bonus`` 는 분명하게 높게 둡니다.
* ``Threat Step Penalty`` 는 중간 이상으로 둬서 SAM 지역 장기 체류를 줄입니다.
* ``Time Cost`` 는 너무 높이지 말고, 느리더라도 살아남는 정책을 허용합니다.
* ``High Value Bonus`` 는 활주로와 지휘 거점을 우선할 때만 높입니다.


배치 최적화에 쓰는 현실적인 방법
-----------------------------

직접 최적화가 아닌 간접 최적화
~~~~~~~~~~~~~~~~~~~~~~~~~~~

현재 구조에서 RL이 직접 배우는 것은 배치 자체가 아니라 ``고정 배치 위 운용 정책`` 입니다.
그래서 배치 최적화는 아래 2단계로 접근하는 것이 가장 현실적입니다.

1. 사람이 배치안 여러 개를 만듭니다.
2. 각 배치안 위에서 같은 RL 실험을 돌려 ``success_rate`` 를 비교합니다.

즉, RL은 "배치 평가기" 역할을 하고, 사람이 "배치 생성기" 역할을 맡습니다.

배치안 A/B 비교 예시
~~~~~~~~~~~~~~~~~~~

한국-북한 시나리오에서는 아래 변수를 바꿔가며 배치안을 만들기 좋습니다.

* ``L-SAM`` 과 ``Cheongung-II`` 의 전방/후방 위치
* 서해 구축함 차단선의 북상 정도
* 드론 ISR 박스의 북쪽 편중 여부
* 전차와 기계화 예비대의 집결 위치
* F-35 중심 편대인지, F-15 중심 편대인지

예를 들면 다음처럼 실험할 수 있습니다.

* A안: 방공망을 깊게 두고, 함정은 후방 보존
* B안: 방공망을 서쪽으로 전진시키고, 구축함 차단선을 북상
* C안: 드론 ISR과 활주로 절단에 더 집중

이후 각 안에서 같은 알고리즘, 같은 표적 묶음, 같은 timesteps로 실험해
최종 ``success_rate`` 와 손실 양상을 비교합니다.

결과 해석 기준
~~~~~~~~~~~~~

* ``success_rate`` 가 가장 먼저입니다.
* ``mean reward`` 는 보조 지표입니다.
* 리플레이에서 발사대 제압 시점이 빨라졌는지 확인합니다.
* 승률은 같은데 손실이 줄었다면 더 좋은 배치일 가능성이 큽니다.
* reward만 오르고 성공률이 그대로면 보상 설계가 과도하게 shaping 되었을 수 있습니다.


시나리오별 권장 질문
-------------------

한국 vs 북한 - 서해 합동 방어
~~~~~~~~~~~~~~~~~~~~~~~~~~~

* 발사대 제압 우선인가, 활주로 절단 우선인가
* F-35 포함 편대가 손실을 줄이는가
* 장거리 SAM을 먼저 누르면 후속 손실이 얼마나 줄어드는가

알다프라 드론 공습
~~~~~~~~~~~~~~~~

* 기지 방어 우선 정책이 실제 승률을 높이는가
* 발사대 억제를 먼저 해야 드론 피해가 줄어드는가
* 수송기와 급유기 보존을 얼마나 중시해야 하는가

호르무즈 봉쇄
~~~~~~~~~~~

* 현재 환경에서는 제한적 참고용으로만 쓰는 것이 좋습니다.
* 해상 차단선 배치 비교보다는, 도서 발사대와 연안 방공 제압 실험 정도에 더 적합합니다.


현재 한계와 다음 단계
-------------------

현재 구조의 한계
~~~~~~~~~~~~~~~~

* 항공 타격 서브태스크에 최적화돼 있습니다.
* 지상 기동전, 해상 maneuver, 드론 초계 박스 자체는 action으로 직접 노출돼 있지 않습니다.
* 그래서 "탱크를 어디 놓아야 가장 잘 이기나"를 RL이 바로 답해주지는 못합니다.

다음 단계 확장 방향
~~~~~~~~~~~~~~~~~

배치 자체를 직접 최적화하려면 새 환경이 필요합니다.
예를 들면 ``ScenarioDeployment-v0`` 같은 환경을 추가해 다음을 action으로 노출할 수 있습니다.

* 방공 포대 앵커 포인트 선택
* CAP 박스 위치 선택
* 드론 초계 박스 선택
* 전차/기계화 예비대 출발 위치 선택
* 함정 차단선 위치 선택

이 경우 reward도 ``고정 표적 타격 성공`` 이 아니라 다음처럼 넓혀야 합니다.

* 아군 핵심 자산 생존
* 적 발사대 억제
* 수도권 또는 기지 방어 성공
* 누수 피해 최소화
* 임무 시간과 손실 최소화
